⚽ Política de Privacidade — Futdle

Última atualização: abril de 2026

O Futdle é um jogo de adivinhação de times de futebol desenvolvido para Android. Esta política explica quais dados coletamos, como os usamos e como os protegemos.

1. Dados coletados

  • Identificador anônimo: ao abrir o app pela primeira vez, um ID de usuário anônimo é gerado automaticamente para salvar seu progresso. Nenhum dado pessoal é necessário.
  • Apelido (nickname): você escolhe um nome que aparece no ranking do desafio diário. Esse nome é armazenado em nossos servidores.
  • E-mail e senha (opcional): se você optar por criar uma conta para salvar seu progresso em nuvem, seu e-mail será armazenado de forma segura via Supabase.
  • Progresso do jogo: número de tentativas, tempo para acertar, desafios jogados e energia disponível.

2. Como usamos os dados

  • Exibir o ranking diário com apelidos dos jogadores
  • Salvar e recuperar seu progresso entre sessões
  • Manter o sistema de energia (vidas) funcionando corretamente

3. Compartilhamento de dados

Seus dados não são vendidos nem compartilhados com terceiros para fins comerciais. Utilizamos os seguintes serviços de terceiros:

  • Supabase (supabase.com) — banco de dados e autenticação. Política de privacidade do Supabase
  • Google AdMob — exibição de anúncios. O AdMob pode coletar dados para personalização de anúncios conforme a política do Google.

4. Retenção e exclusão de dados

Seus dados de progresso ficam armazenados enquanto você usar o aplicativo. Para solicitar a exclusão dos seus dados, entre em contato pelo e-mail abaixo. Usuários anônimos que desinstalarem o app perdem automaticamente o acesso aos dados locais.

5. Segurança

Utilizamos comunicação criptografada (HTTPS/TLS) para todas as transferências de dados. Senhas são armazenadas com hash seguro via Supabase Auth.

6. Crianças

O Futdle não é direcionado a crianças menores de 13 anos e não coleta intencionalmente dados dessas crianças.

7. Alterações nesta política

Podemos atualizar esta política ocasionalmente. Qualquer mudança significativa será comunicada dentro do aplicativo.
